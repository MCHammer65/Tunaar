Run Tunaar on your NAS

Tunaar is one self-contained, multi-arch Docker image (amd64 + arm64) with ffmpeg built in. If your NAS runs containers, it runs Tunaar — Synology, QNAP, Unraid, TrueNAS SCALE, Asustor, Terramaster, OpenMediaVault, or anything with Portainer.

Image: ghcr.io/mchammer65/plexiptv:latest

The one thing to know first: networking

Tunaar advertises an HDHomeRun tuner that Plex/Emby/Jellyfin can auto-discover over a UDP broadcast on port 65001. That broadcast only reaches the container in host networking mode.

ModeWhat you getWhen
Host networking bestAuto-discovery works; your media server finds Tunaar by itself.Unraid, QNAP Container Station, TrueNAS, most Linux NAS.
Bridge + port mapMap TCP 5004; then add the tuner by IP in your media server (http://<nas-ip>:5004). Everything else is identical.Synology and anywhere host mode isn't available.
Both modes stream and serve the guide identically. The only difference is whether the tuner appears automatically or you paste its address once. Always use the URL with the :5004 port.

Reference compose

Every platform below is the same container; this is the canonical definition. Keep /config on a persistent share so your device id and settings survive updates.

services:
  tunaar:
    image: ghcr.io/mchammer65/plexiptv:latest
    container_name: tunaar
    restart: unless-stopped
    network_mode: host            # or: ports: ["5004:5004"] on Synology
    environment:
      - TUNAAR_PLAYLIST=https://iptv-org.github.io/iptv/index.m3u
      # - TUNAAR_EPG_URL=https://example.com/epg.xml
      # - TUNAAR_TUNER_COUNT=4
    volumes:
      - /share/appdata/tunaar:/config

You don't have to set a playlist here — leave it empty and add sources from the dashboard wizard at http://<nas-ip>:5004 instead.

Synology (Container Manager / DSM 7.2+)

  1. Install Container Manager

    From Package Center (older DSM: "Docker"). Create a shared folder for config, e.g. /docker/tunaar.

  2. Create a project

    Container Manager → Project → Create. Paste the reference compose, but Synology doesn't expose host networking on most models, so swap network_mode: host for a port mapping:

        # network_mode: host
    ports:
      - "5004:5004"
    volumes:
      - /volume1/docker/tunaar:/config

  3. Run it

    Build/start the project. Open http://<nas-ip>:5004 and finish the setup wizard.

  4. Add the tuner by IP

    In Plex/Emby/Jellyfin DVR setup, choose enter address manually and use http://<nas-ip>:5004, then point the guide at /epg.xml. (Auto-discovery won't work without host networking — this is the only extra step.)

QNAP (Container Station)

  1. Open Container Station

    Install it from the QNAP App Center if needed.

  2. Create an application

    Applications → Create and paste the reference compose as-is — Container Station supports network_mode: host, so auto-discovery works. Point the volume at a real share, e.g. /share/Container/tunaar:/config.

  3. Start & open

    Create, then browse to http://<nas-ip>:5004 and run the wizard. Plex should find the tuner automatically.

Unraid

  1. Add the template

    Easiest: Docker → Add Container and paste the template URL, or import unraid/tunaar.xml from the repo. It pre-fills the image, the /config path and host networking.

  2. Apply

    Set your config path (e.g. /mnt/user/appdata/tunaar), keep Network Type: Host, and Apply. Open the WebUI link to the dashboard.

Unraid runs host networking by default, so Plex auto-discovery just works.

TrueNAS SCALE

  1. Use the Docker/compose app

    On SCALE (Electric Eel+), use Apps → Discover → Custom App (YAML) or the Docker compose pane, and paste the reference compose. Set the host path for /config to a dataset, e.g. /mnt/tank/apps/tunaar.

  2. Host networking

    Enable host networking in the app's network options for auto-discovery; otherwise map port 5004 and add the tuner by IP.

Portainer (any platform)

  1. New stack

    Stacks → Add stack, paste the reference compose, deploy. Works on any NAS or host where Portainer is installed.

  2. Plain Docker CLI

    Or skip the UI entirely:

    docker run -d --name tunaar --restart unless-stopped \
  --network host \
  -v /share/appdata/tunaar:/config \
  ghcr.io/mchammer65/plexiptv:latest

Health checks

For NAS app managers and orchestrators, Tunaar exposes two endpoints:

EndpointMeaning
/healthzLiveness — the process is up. Used by the built-in Docker HEALTHCHECK.
/readyzReadiness — the lineup has built and Tunaar can serve. Returns 503 on a hard config/load error (a single skipped source is still "ready").
Updating: on any platform, pull the latest image and recreate — your /config volume keeps everything. Optionally run the Watchtower companion to do it automatically.
